This page will teach you on how to format your code properly to make it look organized.

The advantages of having organized code is that you won't be confused when looking through it, allows you to work more efficiently and think more clearly.

Here are some examples of bad code formatting:

Examples of bad formatting:

Hence, making your work a lot more faster than before, spend less time reading and making yourself lesser of a 'beginner' coder.

Common mistakes and error

We can never escape from mistakes in coding. It happens to everyone, even to the professionals. I will list the most common errors in both CSS and HTML.

  • Missing ending tag 
    <script src="script.js">
/// ERROR! The browser cannot find and will not find script.js as there is no ending </script> tag.
  • Incorrect spelling 
    <centre></centre> 
/// ERROR! It is spelled "center" not "centre".
  • Missing bracket 
    .container {
    display: flex;

/// ERROR! There is no ending bracket }. "Container" will not be valid.
