This page will show you the most used CSS elements when building a website, and why you should use them as well.

I often use these, and it's only based on my experience. I won't be getting to too-advanced CSS elements, only the ones discussed.

There are 3 common types of displays:

  • display: block
  • display: flex
  • display: none

The most useful out of all displays is probably display: flex, the second being display: none and the last award goes to display: block.

The reason why display: block is kind off bad is because you can achieve the same results with display: flex with setting the flex-direction to column.

display: flex

Display flex means that items are ordered in a chronological order, and that they are side-by-side or top-and-bottom.

Example:
.container {
    display: flex;
    width: 100%;
    height: fit-content;
}
<div class="container">
  <span style="background: rgba(0,0,0,1); height: 35px; width: 35px;"></span>
  <span style="padding: 10px;"></span>
  <span style="background: rgba(0,0,0,1); height: 35px; width: 35px;"></span>
  <span style="padding: 10px;"></span>
  <span style="background: rgba(0,0,0,1); height: 35px; width: 35px;"></span>
</div>

When we put items on our .container class, items are arranged horizontally (by default) and chronologically.

If we change the flex-direction of our .container class to column, it will look like this:

.container {
  display: flex;
  width: 100%;
  height: fit-content;
  flex-direction: column;
}

No other flex attributes shall be needed when building a website, as they are only "fillers". You don't really need them. All you need is flex-direction CSS attribute.

display: none

It is what it says. display: none literally hides the element. This is useful when on responsive websites, you want to hide that element when resized or in mobile.

display: block

Elements are ordered like a stack/block, each one below of another. Just like on display: flex when set to column.

This website is on display: block. Hence why everything is ordered from top to bottom.

There are 3 common types of position:

  • position: absolute
  • position: relative
  • position: fixed

position: absolute and position: fixed are similar. Both can interchange and you won't see that much difference. However, position: relative is interesting.

For the sake of simplicity, we will only use position: fixed for both absolute and fixed examples.

position: fixed

Like the name says, an element with position: fixed is fixed with respect to the viewport. It stays where it is, even if the document is scrolled.

Let's view an example:

.item {
    position: fixed;
    width: fit-content; 
    height: 35px;
    background: black;
}

As you can see, the fixed element stays where it is positioned.

position: relative

When an element inherits the position: relative css attribute, it means that the element will be positioned relative (or inside) to it's parent container. Elements with position: relative is the same as the default/normal position, but you can use css attributes like top: bottom: left: right: to move the element.

Don't get it? Let's view an example.

Relative examples

.item {
    position: relative;
    width: fit-content; 
    height: 35px;
    background: black;
}

.parent-container {
    border: solid 2px black;
    width: 50px;
    height: 50px;
}
<div class="parent-container">
  <div class="item"><span style="color: white;">I am a relative item!</span></div>
</div>
I am a relative item!

Moving elements

The biggest advantage when an element inherits the position: css attribute is that you can move it freely, anywhere.

Unlike elements with the position: css attribute, you can't move them freely. You would need to go through the hassle of padding and margin trial and errors, and when inspecting the website, it will look very bad and unorganized.

How do we move elements?

We will use 4 css attributes to move elements: top, bottom, left, and right. Yes, those are the names. Let's view some examples:

Left

.item {
    position: relative;
    width: fit-content; 
    height: 35px;
    background: black;
    left: 10px; // This means that our element will be offset/move by 10px from the left 
}

.parent-container {
    border: solid 2px black;
    width: 50px;
    height: 50px;
}
I am a relative item with left: 10px;

Right

.item {
    position: relative;
    width: fit-content; 
    height: 35px;
    background: black;
    right: 10px; // This means that our element will be offset/move by 10px from the right.
}

.parent-container {
    border: solid 2px black;
    width: 50px;
    height: 50px;
}
I am a relative item with right: 10px;

As you can see, our item is past the border. That is normal. Just make sure to use these css attributes properly to avoid that. As with top and bottom, it does the same job as left and right.

Tips and tricks, what to avoid

  • Sidebars and navigation bars should always have position: fixed. Because you don't want them to disappear on-scroll. You want them to be always accessible by the user.
  • Only use position: relative on elements inside a container. You don't want elements to be adjusting and floating around.
  • It is recommended to use position: when applicable. Padding and margin just looks messy.

FAQs:

When to use percentages or pixels?

- Use percentages when you want that container to always stay on screen, and you want that container to auto-adjust and auto-size. For example, if a container has a width of 30%, it's size will shrink with the browser size. Let's view an example:

I am a container with 30% width.

Try resizing the browser. See the different between pixels and percentages.

I am a container with 300px width.

As you can see, the container with the percentage width resizes with accord to the browser size, while the container with pixel width keeps and stays it's width.

Why not include float: ?

For me, you don't really need the float css attribute unless very needed so. You can achieve the same results with using display flex.
